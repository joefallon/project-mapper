Thanks for your interest in contributing to project-mapper!

Basic guidelines
- Open an issue first to discuss larger changes.
- Fork the repository and create feature branches for pull requests.
- Run `npm run typecheck` and `npm test` before opening a PR.
- Write unit tests for new behavior and follow the existing code style in `src/`.

Submitting a PR
- Keep changes focused and document rationale in the PR description.
- Link related issues and provide steps to reproduce when relevant.

Code of conduct
This project follows the `CODE_OF_CONDUCT.md` in the repository. Be respectful in all interactions.
